Welcome to Verman Lab, a portal for visualizing PCI Device support in various kernel versions.


You can either diff kernels or check RHEL6/RHEL7 PCI device enablement.



To add a new kernel:

  • note: the kernel must be installed normally (as the script to parse the kernel and upload to the database looks in /lib/modules)
  • go to the upload page
  • download the shell script on that page
  • upload the .tar.gz file created by the shell script on that page
  • wait for the kernels from that machine to be parsed and added to the database (this may take a while)
    • note: if for whatever reason the load times out, the upload process is still probably working. You can access the admin interface and see if this is the case (kernel versions, modules, and aliases should be actively populating, if they are stalled, please email me).
  • note: if this release is an externally-facing release add the errata information (ie: RHEL 6.2)
    • go to the admin page and login with credentials UN: redhat_rules PW: redhat_is_the_best
    • click "Kernel versions"
    • select the kernel version you wish to change the errata for
    • enter the errata in the text box next to errata

To delete a kernel:

  • note: deleting a kernel will not delete the modules and aliases that kernel uses. this has no functional impact (as you can still reupload the kernel and it will not create duplicate pci modules or aliases in the database)
  • go to the admin page and login with credentials UN: redhat_rules PW: redhat_is_the_best
  • click "Kernel versions"
  • select the checkbox next to the kernels you wish to delete (please don't delete one of the general release kernel versions, such as 6.3, 7.0, etc.)

To update the pci.ids database:

  • note: you should do the following to check to make sure that you are grabbing the correct (new pci.ids) data. At the time of the writing of this application, I wasn't sure if the url would change on the pci.ids side for accessing the latest pci.ids device information. If this part is confusing email me at jlmitch5@ncsu.edu, and I'd be happy to help
    • go to the pci.ids database
    • click "pci.ids" under "Download"
    • if the link directs you to http://pciids.sourceforge.net/v2.2/pci.ids , skip the rest of the substeps
    • if the link is different, Change the link in PROJECT_ROOT/pci_ids/views.py (also edit the above link in these directions to the newer one)
  • go to the pci_ids update page
  • wait for the app to pull the latest pci.ids file, it will display "pci.ids file uploaded to verman lab database."
